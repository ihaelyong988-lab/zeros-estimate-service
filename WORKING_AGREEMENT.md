# 작업 원칙 (Working Agreement)

> 사용자 지시로 기록 (2026-06-20). 이후 모든 작업에 우선 적용한다.

## 1. 의도·맥락 우선 (가장 중요)
- 사용자가 작업을 지시하면 **먼저 "의도(intent)와 맥락(context)"을 파악**한다.
- 화면/이미지/텍스트를 **자세히 보고** 무엇을 원하는지 이해한 다음 움직인다.
- **조금이라도 헷갈리면 임의로 진행하지 말고 먼저 확인받는다.**
- 확인 없이 내 판단대로 작업하면 = **재작업(rework)**. 이는 시간 낭비이자 신뢰 손상이다.
- **지시 문장만 그대로 실행하지 않는다.** 작업 내내 맥락의 디테일(정렬·크기·대비·톤·타깃 고객)을 계속 생각하며 결정한다. 문장에만 반응하면 계속 오류가 난다.

## 2. 시안 먼저, 코드 나중
- 가능하면 코드 반영 전에 **시안(mockup)을 먼저 렌더해 보여주고** 사용자가 고르게 한다.
- "혼자 삽질"하지 않는다. 가진 도구(frontend-design / ui-ux-pro-max / 시각화 / 라이브 프리뷰)를 쓴다.

## 3. 범위 엄수
- 사용자가 "딱 이곳만"이라고 하면 **그 범위만** 작업한다. 마음대로 다른 곳을 건드리거나 되돌리지 않는다.
- 이전에 사용자가 의도적으로 단순화/삭제한 것을 **임의로 되살리지 않는다.**

## 4. 디자인 철학 (이 프로젝트)
- 핵심: **단순 · 간결 · 하이라이트화.** 정보량보다 **위계(hierarchy)**. 다 강조하면 강조가 없다.
- **설명문 = 핵심 내용 + 키워드만(짧게).** 뻔한 라벨·완성형 문장 금지. (예: 큰 ₩숫자 위 "견적 예상가" 라벨 불필요)
- **결과값 = 시각화 지표로.** 숫자 나열 대신 그래프·막대·눈금·아이콘 칩으로 직관화. 중장년 타깃 — 불필요한 영어 절제.
- 눈의 피로를 줄인다: 박스(도형) 남발 대신 여백·헤어라인으로 구분, 색은 핵심 하나에만.
- 단정적 표현 금지(거품·과다청구·부풀려진 단가 등). ZEROS는 **최적합 견적을 산출·제공하는 서비스** 회사다. 시공(공사)하는 곳이 아니다 — 문구 작성 시 항상 유념.
- **직관 우선.** 보통 사람의 뇌는 "생각해서 이해"보다 "보고 즉시 공감"에 끌리고 복잡함을 본능적으로 회피한다. 읽게 만들지 말고 보면 바로 느끼게.
- **중복 금지.** 같은 숫자·문구·절차를 여러 곳에 반복하지 않는다. 각 영역은 **자기 하이라이트 하나만**.
- **키워드는 지표처럼 크게.** 주요 항목은 작은 칩보다 큰 헤드라인(에디토리얼)으로. 폰트는 Pretendard.

## 5. 위임 모드("자동 전환")
- 사용자가 "작업 완료하세요 / 자동 전환합니다"처럼 **명시적으로 위임**하면, 매 항목 확인하지 말고 **도메인 기준 합리적 기본값**으로 진행한다.
- 단, 추후 교체가 쉽도록 **콘텐츠는 데이터(맵)로 분리**해 둔다(공종별 키워드/검토 항목 등).

## 5-1. "마감처리" = 끝까지 (반복 지시 금지)
- 사용자가 **"마감처리"**라고 하면 매번 묻지 말고 **전 과정을 자동 수행**한다:
  **빌드 확인 → 브랜치 커밋 → 푸시 → master 머지 → 푸시(=호스팅 자동 배포)**.
- 기본 브랜치(master)에서 작업했으면 먼저 새 브랜치를 파서 커밋한 뒤 master로 머지한다.
- 배포 메커니즘: 로컬 vercel/deploy 스크립트 없음 → **origin master 푸시 시 GitHub 연동 호스팅이 자동 배포**. (원격: `ihaelyong988-lab/zeros-estimate-service`)
- 커밋 메시지 끝에 `Co-Authored-By: Claude Opus 4.8 <noreply@anthropic.com>`.
- 같은 지시를 반복하게 만들지 않는다 — 한 번 위임받은 마감 플로우는 이후에도 동일하게 수행.

## 6. 오늘 확정된 구조 (2026-06-20)
- **견적 작업 FLOW**: 공종 상세에서 제거 → **견적 검토 랜딩 히어로 박스**에만 1곳.
- **공종 상세 히어로**: 좌=검토 항목(문제 도출)+"최대 N% 견적 차이" / 우=ZEROS Agent AI **견적 검증**+"최적합 견적 확정". **전 공종 공통·데이터 연동**(`tradeReviewItems`, `metrics.bubbleRate`).
- **주요 견적 키워드 밴드**: 공종 상세 최상단, 큰 키워드(B 스타일), 공종별 `tradeKeywords`로 교체.
- **우측 결과 패널**: 견적 검토 진입 시 숨김 → 우측 상단 "검토" 엣지 탭(음영 없음)으로 펼침, 헤더 `›`로 접기.
