# 하네스 · Loop 엔지니어링 규칙 (검증 루프 강제)

> 명문화 2026-06-30. 모든 폴더·모든 프로젝트·모든 작업에 적용되는 표준.
> 동일 내용이 전역 `~/.claude/CLAUDE.md` 와 각 프로젝트 `AGENTS.md`(§11)에 저장돼 있다.
> (폴더명 "Hermer"는 사용자 표기 그대로 — Harness의 표기로 추정.)

---

## 핵심 명제
**"리마인더가 아니라 검증."**
규칙은 말로 된 지시가 아니라 **기계가 채점하는 게이트**로 구현한다.
스킬·도구는 "호출"이 아니라 **"결과를 산출물에 반영"까지가 완료**다.

## 0. 왜 (근본 원인 차단)
- 리마인더(매 지시 재주입)만 있으면 AI는 **검사 가능한 것(=호출)** 만 하고
  **검사 안 되는 것(=결과 적용)** 을 건너뛴다 → 같은 구멍이 반복되고 시간·토큰을 낭비한다.
- 처방: **산출물(코드/문서/데이터)을 기계가 채점**하게 만들고, 미통과면 **마감 자체를 차단**한다.
  "호출만 하고 적용 안 하기"를 구조적으로 불가능하게 만든다.

## 1. 3계층 구조 (새 프로젝트에 이식)
1. **기준파일 (Source of Truth)**
   - UI: `design-system/<proj>/MASTER.md` — `ui-ux-pro-max --persist`로 1회 생성 +
     프로젝트 확정 규칙(색·폰트·대비·금지)을 **최상단 오버라이드**로 고정.
   - 비-UI: 도메인 기준 문서(스키마·수식 규칙·체크리스트).
   - 매번 재추론하지 말고 이 파일을 따른다.
2. **검증 루프 (핵심 · Stop 훅 차단형)**
   - 이번에 바뀐 산출물을 **결정적 룰(grep/lint/test)** 로 채점.
   - **위반 또는 미검증이면 Stop 훅이 `decision:block` 으로 마감을 막고 체크리스트를 재주입 → 통과까지 loop.**
   - 무한루프 방지: 세션당 차단 3회 초과 시 경고 후 통과(사람 판단).
3. **강제 게이트 (선택 · PreToolUse)**
   - 산출물 편집 전 해당 스킬 호출 마커가 없으면 편집 차단.

## 2. 작동 규약
- 산출물 변경 시 **마감 전**:
  `node .claude/hooks/ui-quality-gate.mjs --check` → 위반 수정 →
  `--pass`(차단룰 0건일 때만 통과 마커 기록). 미통과면 Stop 게이트가 마감을 차단.
- UI 차단/경고 룰 예시:
  - 에러 메시지 `role="alert"`/aria-live (차단)
  - 본문 대비 4.5:1 — 저대비 회색 본문 금지 (경고)
  - 터치 타깃 ≥44px (경고)
  - `focus-visible` 가시 · 이모지 아이콘 금지 · `prefers-reduced-motion` 존중 · 숫자 `tabular-nums`
- 비-UI(업무문서·데이터)도 동일 패턴: 기준파일 + 결정적 검증(스키마/수식/테스트) + 미통과 시 마감 차단.

## 3. 이식 의무
- 새 폴더/프로젝트 착수 시 `harness-setup`으로 3계층(기준파일·Stop 검증 루프·강제 게이트)을 이식.
- 이 폴더의 `templates/` 를 복사해 빠르게 깐다(아래 설치 절차).
- 마감 자기점검에 **"이번 산출물이 검증 루프(--pass)를 통과했는가"** 포함.

## 설치 절차 (새 프로젝트에 깔기)
1. `templates/ui-quality-gate.mjs` → 대상 프로젝트 `.claude/hooks/ui-quality-gate.mjs` 로 복사.
2. `templates/settings.stop-hook.snippet.json` 의 Stop 항목을 대상 `.claude/settings.json` 의 `hooks.Stop[0].hooks` 배열에 추가(기존 훅과 병행).
3. UI 프로젝트면 `ui-ux-pro-max --persist -p "<프로젝트>"` 로 `design-system/<proj>/MASTER.md` 생성 후 확정 규칙 오버라이드 추가.
4. 동작 확인: 산출물 일부를 의도적으로 위반시켜 `--check` 가 잡는지 검증.

## 검증된 사례 (ZEROS, 2026-06-30)
- 견적문의 탭 재설계 시 ui-ux-pro-max를 "호출만" 하고 접근성(에러 alert·대비·터치 44px)을 미반영한 구멍 발생.
- 검증 루프 도입 후 `--check` 가 R1/R2/R3를 즉시 검출 → 수정 → `--pass` → 위반 0건으로 마감.
