## 1. 풀스크린 스냅 2페이지 구획 + 라벨 정정 — `7fcc79e`

**문제**: 휴대폰에서 히어로 + 분석 대시보드가 한 스크롤에 흘러들어 하단 카드가 잘려 보임.

**변경**
- 모바일 랜딩을 **두 개의 풀스크린 스냅 페이지**로 분리.
  - 스크롤 컨테이너에 `snap-y snap-mandatory`(`AppShell.tsx`, `isMobileLanding` 분기).
  - 각 섹션 `min-h-full snap-start` → 밀면 페이지 단위로 스냅.
  - 1페이지 = 히어로 / 2페이지 = 현재 프로젝트 분석 + 실시간 분석 현황.
- 히어로에 **핵심 3대 역량 아이콘 칩** 신설: 무료 출장 견적 서비스(`Truck`) / AI 데이터 분석(`LineChart`) / 현장 실무 30년(`Award`).
- 하단 "현재 프로젝트 분석 ↓" 스와이프 큐 추가.
- **라벨 정정**: `배관 공사` → **총 공사 견적금액**, `예상 가격` → **예상 견적**.
- 데스크탑(`lg`) 레이아웃 무영향.

## 2. 견적 슬라이더 드래그 연동 복원 — `1e431d4`

**문제**: "실시간 분석 현황" 막대가 정적 그래픽(핸들 `left-[45%]` 고정, 금액 상수)이라 움직이지 않음.

**변경**
- 상태를 컴포넌트 본문으로 승격: `const [mobileEstimateAmount, setMobileEstimateAmount] = useState(26850000)`.
- 투명 `<input type="range">`(min 12,000,000 / max 45,000,000 / step 50,000)를 트랙 위에 덮어 **터치·드래그로 값 조절**.
- 핸들·"예상 견적" 라벨 위치 = `(값-최소)/(최대-최소)` 로 실시간 이동.

## 3. 짤림 해소 — dvh 뷰포트 + 히어로 높이 압축 — `2366b74`

**문제**: 첫 페이지 하단(CTA·스와이프 큐)이 잘리고 마지막 페이지가 안 보임.

**원인**: 모바일 컨테이너가 `h-screen`(=100vh). 모바일 브라우저는 **주소창 높이까지 100vh에 포함**하므로 레이아웃이 실제 가시영역보다 커져 하단이 잘림.

**변경**
- `AppShell.tsx`: `h-screen` → **`h-[100dvh]`**(동적 뷰포트) — 주소창 유무와 무관하게 실제 가시영역에 정확히 맞춤.
- 히어로 높이 압축: `pt-8→pt-5`, 상단 블록 `gap-7→gap-5`, 칩 `py-3.5→py-2.5`·아이콘 `w-11→w-10`, CTA `min-h-14→min-h-12`, 하단 블록 `pt-6→pt-4`. 제목 `text-[31px]→text-[28px]`.

**검증**
- 720px(주소창 노출 가정): 페이지1 콘텐츠가 스크롤 영역(655) 안(639)에 들어옴 — 잘림 없음.
- 915px: 2페이지 정확히 스냅, 마지막 "서비스 수수료" 행까지 완전 노출(801 ≤ 850).

## 7. 3페이지화 + 화살표 탭 네비 + 또렷한 스냅 — `483748b`

**요청**: 랜딩을 총 3페이지로, 화살표 탭 2개 또는 스와이프(딸깍)로 이동.

**변경**
- 기존 2페이지(현재 프로젝트 분석 + 실시간 분석 현황)를 **둘로 분리**:
  ②`m-landing-2` = 현재 프로젝트 분석(지표 카드) / ③`m-landing-3` = 실시간 분석 현황(슬라이더).

- **↓ 화살표 탭 2개**: 1→2(라벨 "현재 프로젝트 분석"), 2→3(라벨 "실시간 분석 현황").
  `document.getElementById('m-landing-N').scrollIntoView({behavior:'smooth',block:'start'})` — 외부 의존성 없음.
- 스냅 `proximity` → **`mandatory`** 복귀(또렷한 '딱'). 외곽 높이 `dvh` → **`svh`**(랜딩 한정,
