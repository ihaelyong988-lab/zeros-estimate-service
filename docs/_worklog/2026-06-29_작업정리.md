# 2026-06-29 작업정리

## PHASE O-8 — 견적문의 탭 재설계 (로그인 게이팅 + 2채널 자료등록 + 예약방문 신규)

### 배경 / 지시 (모바일 캡처 7개 항목)
사용자 캡처 주석을 전수 번호화하여 처리. "우선 복잡한 것보다 단순화 필요".

| # | 지시 | 결과 |
|---|---|---|
| ① | 상단 검은 설명 박스 삭제 | 헤더 설명문구 제거·단순화, 채널 인지 제목으로 교체 |
| ② | 한 번 로그인된 사용자는 자동으로 자료등록 화면만 | `customerAuth`면 게이트·박스 없이 바로 2채널 자료등록 화면 |
| ③ | 처음 사용자 로그인 등록 → 자료등록 화면 | OTP 인증 게이트 → 인증 시 **로그인 영속화**(다음 방문 자동 통과) |
| ④ | 견적·출장요청 = 자료등록 + 예약방문 신청 코너 | 채널 A: 자료 업로드 + 희망 방문일·시간대 → `zeros_site_visits` 기록 |
| ⑤ | 무료 견적(총 공사비 100만원 이하) = AI Native 자동 등록 탭 | 채널 B: `estimate_category:'small'`·`≤1,000만` 자동 분류 |
| ⑥ | 견적문의 탭 = 사용자 등록 + 자료 등록 코너 | 로그인 게이팅 + 2채널 선택 허브로 재구성 |
| ⑦ | 모든 자료는 이력관리에 저장 | 접수=`zeros_estimates`, 예약=`zeros_site_visits` 동시 저장 |

### 진행 방식
- AGENTS.md §0대로 `ui-ux-pro-max`(design-system) 선호출 → Pattern=Lead Magnet+Form, Style=Trust & Authority 확인(기존 navy+오렌지 톤과 일치).
- §2대로 `show_widget` 모바일 시안 3화면(처음 사용자 로그인 게이트 → 로그인됨 2채널 → 예약방문 폼) 먼저 제시.
- 구조 결정 3건 `AskUserQuestion`으로 확정: **2채널 선택형 / 예약방문 날짜·시간 폼 신규 / 기존 휴대폰 OTP 사용**(모두 추천안).
- §5 위임("나의 컨셉을 숙지해서 작업해줘")으로 ZEROS 컨셉 적용해 구현.

### 변경 파일
- `components/forms/RequestWizard.tsx`
  - `RequestChannel`('visit'|'quick') 타입 + `initialChannel`/`onChannelConsumed` props 추가.
  - `channel` 상태: null=2채널 선택 화면(자료등록 화면), 값=해당 폼. 헤더에 ← 뒤로(채널 재선택).
  - 헤더 설명 박스 span 삭제(①), 채널 인지 제목(headerTitle).
  - `handleVerified`에서 `setCustomerAuth(...)`로 처음 사용자 인증을 로그인으로 영속화(②③).
  - 2채널 선택 카드 UI(Truck=출장요청 / Zap=무료견적), steel 단일 강조·중장년 가독.
  - `defaultFormData`에 `visit_date`/`visit_time` 추가, visit 채널 전용 예약방문 입력란(날짜+오전/오후) + 필수 검증.
  - `handleSubmit` 채널별 메타: quick→small/≤1,000만, visit→예약방문 동반. visit는 `createSiteVisit`로 이력 동시 기록(⑦). 제출 버튼 라벨 채널별.
- `app/page.tsx`
  - `RequestChannel` import, `pendingRequestChannel` ref + `openRequestChannel(ch)` 헬퍼.
  - `renderRequestTab`에 `initialChannel`/`onChannelConsumed` 전달(탭바 재진입 시 선택 화면 복귀).
  - 홈 모바일 카드 "견적·출장요청 자료 등록하기"→채널 A, CTA "무료 견적 신청하기"·서비스소개 "무료 견적 신청"→채널 B로 직접 진입(⑤⑥).
