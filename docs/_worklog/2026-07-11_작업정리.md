# 2026-07-11 작업정리

## PHASE O-26 — 모바일 실적 상단 밴드 수평 스와이프

### 지시·범위

주인님 캡처 기준으로 휴대폰의 `실적 분포` 제목과 KPI 4종이 두 줄로 감기며 세로폭을 차지하는 문제만 수정한다. 하단 실적 히트맵과 같은 네이티브 가로 이동을 적용하고, 차트·히트맵·세부 실적·하단 탭·데스크톱 레이아웃은 변경하지 않는다.

### 반영

1. `components/PerformanceInsights.tsx` 상단 밴드의 `flex-wrap`을 제거했다.
2. 내부에 `overflow-x-auto no-scrollbar`와 `min-w-full w-max` 행을 두어, 모바일에서 `실적 분포 → 견적 건수 → 검토 비율 → 평균 소요 → 공종 수`가 한 줄로 유지되고 손가락으로 좌우 이동한다.
3. 제목과 KPI 클러스터에는 `shrink-0`·`whitespace-nowrap`를 적용했다. 데스크톱은 `ml-auto`로 기존 KPI 우측 정렬을 유지한다.
4. `AGENTS.md`에 같은 모바일 불변식을 기록해 다음 작업에서 세로 줄바꿈이 되돌아가지 않게 했다.

### 인증문자 점검 (코드 변경 없음)

- 간편 로그인/등록은 이미 `/api/otp/send` → Solapi 어댑터 → `/api/otp/verify` 구조다.
- 실서비스 `otp/status`는 비활성 상태이며, 로컬 Solapi 변수도 비어 있다. Solapi의 발신번호 등록·API 키/시크릿·서버 전용 환경변수 등록과 재배포가 필요하다.
- 현행 Solapi 공식 메시지 API의 `send-many/detail` 요청 형식과 응답 실패 목록을 어댑터에 반영한 뒤 활성화해야 한다. 보안 보강(무작위 코드·재발송/시도 제한·서버 토큰 검증)은 별도 승인 범위다.

### 검증

- `npx eslint components/PerformanceInsights.tsx` 통과.
- `npm run build`는 샌드박스에서 컴파일 후 worker `spawn EPERM`으로 중단됐으나, 권한 확장 재실행에서 TypeScript·정적 페이지 생성까지 0 오류 통과.
- `git diff --check` 통과.
- 전체 `npm run lint`는 기존 파일(`app/page.tsx`, `MyRequestsModal.tsx`, `MyRequestsView.tsx`, `AppShell.tsx`, `estimate-ops`)의 React hook/any 오류로 실패했으며 이번 변경 파일에는 오류가 없다.

### 오류 원장

- Windows 샌드박스가 `apply_patch`의 분할 writable-root 준비를 거부했다. 처방: 패치 내용은 먼저 `git apply --check`로 검증하고, 승인된 최소 범위 `git apply`로만 반영한다.
