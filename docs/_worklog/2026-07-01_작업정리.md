## PHASE O-11 — AI Native 검증 SOP "내림차순 깔때기 인포그래픽" 최적화

### 배경 (의도)
- 사용자 캡쳐 ②의 손주석: "전체 내용을 인포그래픽 내림차순으로 직관 시각화 / 한글·영어 이중설명 중 하나만 / 폰트크기·하이라이트로 리듬감 / 설명문 함축 / 주제·절차·결과지표 재정립(중학생 눈높이) / 불필요한 아이콘·이중설명 삭제".
- 직전 PHASE O-10에서 영어·엔진·다크콘솔은 이미 제거됐으나, **funnel 폭 수렴이 97→85%로 거의 안 보이고(=그냥 6줄 리스트) 폰트 크기가 균일(18px)해 리듬감이 없었음**. 캡쳐는 품질 기준선이라 판단해 시각 임팩트를 재설계.

### 한 일
1. **진짜 내림차순 깔때기**: 폭 수렴을 또렷하게 — 모바일 완만(100→82%)·데스크톱 강하게(100→60%). `funnelW` 배열을 `w-[..] md:w-[..]` 페어 클래스로 교체하고, 컨테이너를 `items-center`로 중앙 정렬해 깔때기 형상이 눈에 들어오게.
2. **폰트·하이라이트 리듬감**: 단계별 `size` 필드 도입 — 핵심 단계(거품 탐지·검토서 발행)만 22px로 키우고 나머지 18px. 하이라이트는 거품탐지=accent(오렌지), 검토서=navy 강조 2곳만.
3. **설명문 함축(중학생 눈높이)**: 보조설명을 문장형 → **6~8자 키워드**로 압축("도면을 읽어 자재·물량을 뽑습니다" → "자재·물량 추출"). 결과칩(중복 요소) 제거해 한 줄에 [순번·키워드·함축] 만 남김.
4. **주제·결과 정리**: 헤드라인 "AI가 6단계로 걸러, 거품 없는 적정 예산까지" / 흐름 한 줄 "도면 한 장 → 자동 분석 → 전문가 확인 → 검토서". 결과지표 4종(예산밴드·리스크·범위·근거추적) 유지.
5. **접근성**: 깔때기 순번 색을 `text-gray-light` → `text-gray`로 상향(대비 4.5:1). 이모지 0·아이콘 0(타이포·폭·색만으로 위계).

### 검증
- `ui-ux-pro-max --design-system`: Funnel(3-step)·Trust & Authority·navy+blue CTA·이모지 금지 확인(기존 ZEROS 토큰과 일치).
- UI 품질 게이트 `--check`/`--pass`: 차단룰 R1 0건 통과(R2는 기존 KPI 라벨 경고, 비차단).
- `npm run build`: 0에러. `show_widget` 시안으로 깔때기 형상·리듬 확인.

### 범위 외(의도적 미변경)
- 견적문의 위저드(캡쳐 ①): O-10에서 이미 채널선택→단계탭→3스텝→관리페이지 링크로 충족·검증됨. 작동 중인 폼 검증 흐름을 건드리지 않기 위해 범위 규율(§3)로 미변경.

### 파일

### 배경/의도

- `ui-ux-pro-max`(Trust & Authority) 선호출 — 안티패턴 "AI 퍼플/네온 그라디언트" 제거를 근거로 화이트+네이비+스틸블루+오렌지 단일 CTA 팔레트 확정. `show_widget` 시안으로 매 단계 확인.

### 한 일
1. **①-A 막대↔히트맵 색 통일** (`components/PerformanceInsights.tsx`): 분포 막대 8색 무지개(`TRADE_COLORS`) → 히트맵과 동일 단일 스틸블루 `#1E4D8C`. 미사용 상수 제거.
2. **①-B 우측 예상견적 패널 자동열림 중지** (`components/layout/AppShell.tsx`): 접힘 조건에 `selectedMenu`·`selectedBudget` 추가 → 공종/규모 진입 시 닫힌 채 시작, 우측 "검토" 엣지 탭으로만 펼침.
3. **③ 모바일 메인 화이트 전환** (`app/page.tsx` 3개 스냅 섹션 전부): 다크네이비 `#041B33`→화이트 `#FFFFFF` / 네온 3색 아이콘(sky·indigo·emerald)→스틸블루 `#1E4D8C` 단색 / 반투명 카드→화이트+헤어라인 `#E4EAF2` / 주 CTA→오렌지 솔리드 `#E0701A` 하나 / 견적 카드·슬라이더 라벨 네이비 전환. 템플릿·레이아웃·문구 유지.
4. **④ 정렬 언밸런싱 교정** (`app/page.tsx` 히어로): 헤드라인 `text-center`→좌측 + 칩 `pl-1` 제거 → eyebrow·헤드라인·칩·CTA 동일 좌측 기준선. 섹션 2·3처럼 eyebrow+제목 구조로 3섹션 통일(세로는 중앙 유지).
5. **②·⑤ SOP → 데이터 분석 파이프라인 재구축** (`app/page.tsx` `renderSopTab`): 얕은 6단어 키워드를 폐기하고, 표준 데이터 분석 생애주기(수집→**검증·정제**→전처리→모델링→분석→**검증**→전달)를 반영한 **6단계 수직 파이프라인**으로 교체. 각 단계에 방법론 칩 + 데이터 변환(io) 노출:
   - ①데이터 수집(Multi-source) ②**데이터 검증·정제**(적합성·품질 검증·Gap 보완) ③AI 도면 판독·정형화(Vision AI·OCR) ④데이터 모델링(Feature Engineering) ⑤AI 실거래 벤치마킹·**이상치 검증**(실거래 DB 대조·Z-score, 오렌지 강조) ⑥**결과 검증**·검토서 발행(신뢰구간 ±5%·전문가 검증).
   - 수직 스파인 색 흐름 steel→accent→success. 지표 스트립: 실거래 표본 246건·AI 신뢰도 98.4%·예산 밴드 ±5%·리스크 LOW·근거 추적 100%.

## PHASE O-13 — 실적 대시보드 세련화: 작업 특성 기반 공종 팔레트 + 대시보드 폴리시

### 배경/의도
- 사용자: 실적 집계표 막대·히트맵을 "작업 특성에 맞는 색으로 세련되게" 컬러링, 이어서 "스킬 활용해 최고로 세련되게".
- 직전(O-12)에 막대를 단일 스틸블루로 통일했으나, 사용자가 원한 것은 **공종별 의미 기반 색 + 히트맵 코디**였음(색 통일=매치의 오해). 반영.
- `ui-ux-pro-max`(color·Executive/Data-Dense Dashboard) 선호출 근거로 팔레트·대시보드 폴리시 결정. 스크린샷의 옛 "실적 집계표" 제목+박스 KPI는 O-7에서 이미 제거(캐시 화면)임을 확인.

### 한 일 (`components/PerformanceInsights.tsx`)
1. **작업 특성 기반 공종 팔레트 복원**(`TRADE_COLORS`, 색상심리): 배관공사=블루 · Utility=스틸블루 · 기계실개선=틸 · 장비설치=그래파이트 · 공장증설=앰버 · 노후배관교체=러스트 · 생산설비 배관연결=인디고 · CAPEX=딥네이비. 딥·뮤트 톤으로 통일해 8색이 함께 있어도 정돈·전문.
2. **막대 그래프 컬러링**: 각 공종 시그니처 색(fillOpacity 0.92).
3. **히트맵 코디**: `heatCellStyle(v,max,baseHex)`로 각 행이 자기 공종 색·농도. 행 라벨에 3px 컬러 엣지. 글자색은 **흰 배경 alpha 블렌딩 밝기 기반**으로 산출(밝은 앰버·어두운 네이비 모두 대비 확보). 범례 "셀 색=공종별"로 갱신.
4. **막대 폴리시**: `CartesianGrid` 제거·X축 hide·Y축 라인 제거·뮤트 → **막대 끝 `LabelList` 값 라벨** 직접 표기(에디토리얼). radius 확대.
5. **세부 실적 카드 코디**: 상단 보더·닷·% 칩을 공종 색으로(칩 글자는 대비 안전한 네이비 유지).

- `show_widget` 시안 3종(정제 팔레트·의미 매핑·대시보드 폴리시)으로 확인.
