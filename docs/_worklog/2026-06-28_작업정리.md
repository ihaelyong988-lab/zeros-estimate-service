## §1. 고객 화면 색 위계 청색화 + 공종 밴드 통일 (PHASE O-3)

사용자 캡처 주석 3장 기준. 메인화면 "AI 분석"과 동일한 청색(`#155EEF`)으로 강조색을 통일하고, 공종 상세 밴드를 견적총액별 방안에도 확장.

### 변경 내역 (항목별)
1. **사업소개 히어로 오렌지 → 청색 `#155EEF`** (`app/page.tsx` renderBusinessTab)
   - 헤드라인 "데이터로" (`text-accent` → `text-[#155EEF]`)
   - "안심 예산 대역 ±5%" — 텍스트·세로막대(`bg-accent`)·밑줄(`bg-accent`) 3요소 모두 청색
   - 푸터 "최적합 견적 검증" 라벨 청색
   - **CTA "무료 견적 검토 신청" 버튼은 오렌지 유지** (행동 유도 1곳만 강조 — AGENTS.md §10)
2. **공종 상세 밴드 라벨 `WORK_ITEM` → `주요 공종`** (`app/page.tsx` renderManualDetail, 약 1534행)
   - 색은 기존 청색 `#155EEF` 유지. 공용 컴포넌트라 전 공종 페이지에 자동 반영.
   - 영어 내부 토큰 → 한글 라벨(AGENTS.md §4 영어 절제 부합).
3. **견적총액별 방안에 동일 밴드 추가** (`app/page.tsx` TRADE_KEYWORDS, 약 91~103행)
   - `small`(온라인 간편검토)=소규모 분기·펌프·밸브 교체·단가 검토
   - `medium`(출장견적)=현장 실측·중규모 배관·계통 검토
   - `large`(프로젝트 사전진단)=대규모 설비·CAPEX·공법·예산
   - `renderManualDetail(selectedBudget)`가 small/medium/large로 호출되므로 키워드만 추가하면 동일 밴드가 자동 생성(밋밋함 해소).

### 의미축(색) 규칙 — 유지 필수
- **파랑 `#155EEF`** = 데이터·AI·검증 / **오렌지** = 행동(CTA)·주의(최대 N% 견적 차이)·비용(수수료 2%) / **네이비** = 본문.

### 검증
- `ui-ux-pro-max` 사후 검증: 권장 팔레트(Trust & Authority = Navy + Blue CTA)에 부합. `#155EEF` on 흰배경 대비 ≈ **5.4:1** → 작은 글씨(푸터 12.5px)도 4.5:1 통과.
- `npm run build` 0에러. `graphify update .` 완료(172 nodes / 41 communities).

### 교훈
- UI 작업은 "단순해 보여도" 착수 **전** `ui-ux-pro-max` 호출이 §0 필수 — 이번에 색·라벨 변경을 단순작업으로 보고 건너뛴 뒤 사용자 지적으로 사후 검증함. 다음부터 선(先)스킬.

## §2. 모바일 상단 헤더 정리 + 실적 집계표 KPI 압축 (PHASE O-4)

사용자 캡처 1장(빨간 주석 + X표시) 기준. 모바일 앱 화면의 군더더기 제거 + AI NATIVE 배지 링크화 + 실적 KPI 레이아웃 압축. "깔끔·직관·단조" 추구.

### 변경 내역 (캡처 6항목 전수)
1. **상단 "마이페이지" 칩 삭제** (`components/layout/AppShell.tsx` 약 428행)
   - 로그인 후 표시되던 상단 우측 `마이페이지` 칩 제거 — 하단 탭 `마이페이지`와 중복(AGENTS.md §4 중복 금지). 비로그인 시 `간편 로그인/등록` 버튼은 진입점이라 유지.
2. **AI NATIVE 배지 → 설명 링크 연결** (`AppShell.tsx` 약 451행)
   - 기존 비클릭 `<span>` → `<button>`. 클릭 시 `setActiveTab('sop')`(= 'AI Native 검증' 표준작업절차)로 이동. 모바일 탭은 '서비스 소개'에 매핑(page.tsx 258~260행 동기화 로직).
3. **AI NATIVE 옆 점(녹색 닷) 아이콘 삭제** (동상)
   - `bg-success animate-pulse` 닷 제거(단조·직관).
4. **실적 집계표 설명 단락 삭제** (`components/PerformanceInsights.tsx` 약 158행)
   - "홍보용 과장 수치 대신…" `<p>` 제거, 제목 `ZEROS 실적 집계표`만 유지.
5. **KPI 보조 3종 라벨 단축 + 포지션 압축** (`PerformanceInsights.tsx` 약 166행)
   - 1차에 통째 삭제했다가 사용자 정정("X는 글자 단축·포지션 축소 의미")으로 **복구**.
   - 라벨: `검토 완료율→검토 비율` · `평균 검토 소요→평균 소요` · `진단 공종 수→공종 수`.
   - 크기 축소: 숫자 30→23px, 단위 16→13px, 간격 `gap-x-9→gap-x-6`. 누적 진단 건수(지배지표·유일 컬러)는 유지.

### 검증
- `npm run build` 0에러. `graphify update .` 완료(172 nodes / 41 communities).
- 시각 검증은 시안(`show_widget`)으로 대체(헤드리스 프리뷰 스플래시 정지 — AGENTS.md §8).

### 교훈
- **캡처의 X표시 ≠ 항상 "삭제"**. 이번 KPI는 X가 "글자 단축·포지션 축소"를 의미했는데 통째 삭제로 1차 오독 → 재작업. 다항목 캡처는 애매하면 "이 X가 삭제인지 축소인지" **먼저 확인**(§1-A). 라벨처럼 보여도, 동작 의미도 임의 단정 금지.
