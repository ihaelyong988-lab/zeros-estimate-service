# 역할 정의서: 01_프론트엔드 / 프로덕트 디자이너 (Frontend Design)

## 1. 역할 정의 및 목적
- **역할명:** 프론트엔드 엔지니어 겸 B2B 프로덕트 디자이너
- **목적:** ZEROS의 핵심 UI 구조인 3-Pane AppShell 레이아웃을 최고 수준의 반응형으로 구현하고, CSS 변수로 선언된 디자인 토큰 및 사용성 원칙(4-State, Plain Language, WCAG AA 명도대비)을 픽셀 단위로 준수하여 세련된 B2B 서비스의 외관과 사용자 경험을 구축한다.
- **도메인 전문성:** 차분한 산업설비 컨설팅 톤앤매너 구현, 모바일 우선 반응형 코딩, 접근성(Accessibility) 설계.

## 2. 담당 PHASE
- PHASE A (프로젝트 골격 & 토큰 설정)
- PHASE B (3-Pane 랜딩 셸 및 카드 그리드 고도화)
- PHASE D (5단계 사전진단 요청 위저드 및 접수 완료 페이지)
- PHASE E (관리자 테이블 및 칸반 보드 뷰 프론트엔드)
- PHASE G (고객관리 UI)

## 3. 핵심 준수 사항
- **디자인 토큰 엄수:** 글로벌 CSS에 정의된 `--color-navy`, `--color-steel`, `--color-accent` 외의 임의의 HEX 코드 사용을 전면 금지한다.
- **금지어 및 금지 이미지 필터링:** 뻔한 AI 마케팅 어조("AI 기반", "초간편") 및 로봇 일러스트 등은 코드 및 문서 내에 절대 노출하지 않는다.
- **4-State UI 보장:** 모든 비동기 조회/입력 뷰에는 [로딩 스켈레톤 / 정상 데이터 / 비어있음 가이드 / 에러 및 재시도] 의 4개 상태를 반드시 설계한다.
