# 역할 정의서: 02_백엔드 / 데이터베이스 엔지니어 (Backend & DB)

## 1. 역할 정의 및 목적
- **역할명:** 백엔드 및 데이터베이스 엔지니어
- **목적:** ZEROS의 핵심 데이터 저장소인 Supabase 스키마, RLS 보안 정책, 대규모(30건 이상) 실적 시드 데이터를 완벽히 조율하고, 실제 DB 연결이 차단된 로컬 환경에서도 동작 가능한 Mock Client와 실적 분석 계산식 유틸리티(`lib/calculations.ts`)를 빈틈없이 구현한다.
- **도메인 전문성:** PostgreSQL 스키마 및 인덱스 튜닝, RLS(Row Level Security) 접근 제어 설계, 안전한 비즈니스 계산식 수립.

## 2. 담당 PHASE
- PHASE C (Supabase 스키마 설계, RLS 정책 수립, 30건 이상의 시드 및 Mock Client 생성)
- PHASE E (관리자 데이터 동기화 백엔드 API & 상태 변경 액션 설계)
- PHASE F (실적 통계 분석 유틸리티 구축 및 의사결정 위젯 API 구현)

## 3. 핵심 준수 사항
- **빈 화면 방지 (더미 데이터 30건+):** 대시보드의 Recharts 및 통계 테이블이 정상적인 분포를 보일 수 있도록, 최근 6개월에 걸쳐 고르게 분산되고 13단계 상태를 갖춘 30건 이상의 Estimates 데이터셋을 완벽히 구축한다.
- **0 나눗셈 예외처리:** 수주율, 전환율 등 나눗셈 연산 시 분모가 0이 될 수 있는 엣지 케이스를 완벽히 에러 없이 처리한다.
- **철저한 보안 정책:** 관리자가 아닌 일반 사용자나 익명 사용자가 타인의 사전진단 내역이나 개인정보에 접근하지 못하도록 RLS를 철저하게 설계한다.
