# 프로젝트 의사결정 로그 (Decisions Log)

## [2026-06-02] 로컬 우선 개발을 위한 Mock Client 구조 도입 결정
- **배경:** 사용자의 원클릭 프롬프트 요구에 따라 Supabase DB 자격증명이 없는 로컬 환경에서도 빌드 및 시나리오 테스트가 100% 정상 작동해야 함.
- **결정:** `lib/supabase/client.ts`에 환경변수 `NEXT_PUBLIC_SUPABASE_URL` 및 `NEXT_PUBLIC_SUPABASE_ANON_KEY`가 없을 경우 `MockSupabaseClient`를 주입하는 팩토리 구조를 구현하기로 결정함.
- **기대효과:** 키 없이 `npm run dev` 만으로 전체 어플리케이션이 견적 신청, 접수, 파이프라인 관리, 통계 차트 렌더링까지 완벽하게 구동 가능하여 실패 원인 4번(내용 비어있음)과 2번(기능 미작동)을 전면 해결함.

## [2026-06-02] 3-Pane AppShell 구조 및 프론트엔드 프레임워크 선택
- **배경:** `npx create-next-app` 명령어 옵션을 통해 TypeScript, App Router, Tailwind CSS를 포함한 뼈대를 선택함.
- **결정:** shadcn/ui의 기본 컴포넌트 프리셋을 사용하는 대신, CSS 변수를 완벽히 활용해 Vanilla Tailwind 기반으로 직접 3-Pane 레이아웃과 B2B 스타일 컴포넌트(사이드바, 카드, 테이블, 칸반)를 세밀하게 드로잉하기로 결정함. (스타일 일관성 및 파일 관리 용이성 향상)

## [2026-06-02] 최상단 박싱 규격 단일화 (Phase L)
- **배경:** 랜딩 대시보드와 카테고리 상세 페이지의 최상단 패널이 하단의 카드들에 비해 약간 과도한 마진이나 분산된 박스 구성을 가지고 있어 시각적인 안정감과 디자인 일관성을 저해한다는 피드백 수렴.
- **결정:** 최하단 카드 규격(`bg-bg border border-border p-6.5 rounded-custom shadow-custom-sm`)에 정확히 맞추어 최상단 배너 및 공종 소개 애니메이션 영역 전체를 단일 화이트 B2B 박스로 통합 구현함.
- **기대효과:** 3-Pane 대시보드 전체의 그리드 대칭성이 완전하게 정립되었으며, 모바일과 데스크톱 템플릿 환경 양쪽에서 공통으로 적용 가능한 최상의 통일감 있는 B2B 브랜드 아이덴티티가 수립됨.
