## [2026-06-16] '서비스 프로세스' 탭 신설 (고객 여정 9단계)
- **배경:** 모바일 메인 히어로의 "서비스 프로세스 보기" 버튼과 "현장실무 경력30년 암묵지" 칩이 둘 다 `about`(데이터분석 절차) 탭으로 연결되어 세부 내용이 동일했음. "서비스 프로세스"라는 이름에 맞는 고객 관점의 별도 화면이 부재.
- **결정:** 신규 `process` 탭을 추가(`ShellContext.ActiveTab`, `AppShell` URL/모바일 하단탭 동기화, `app/page.tsx renderServiceProcessTab`)하고, "서비스 프로세스 보기" 버튼만 `about` → `process`로 분리. 데이터 사이언스 파이프라인 언어로 고객 여정 9단계를 정립하되 단계마다 **주체 칩(고객/ZEROS/공동)·산출물 배지·"고객 궁금증 해소" Q&A**를 병기. 3국면(접수&정의→모델링&AI검증→시뮬레이션&전달) 색 흐름(steel→accent→success)으로 진행감 부여. 기존 `sop` 타임라인 디자인 언어 재사용으로 톤 통일.
- **기대효과:** "암묵지" 칩과 "서비스 프로세스"가 서로 다른 가치를 전달(전자=신뢰·자격, 후자=고객이 겪을 절차)하게 되어 정보 중복 제거. 신청 전 단계별 의문을 선제 해소하여 전환 신뢰도 향상. 데스크톱 헤더 IA는 5개 유지(요청 한정이 모바일 버튼이므로 헤더 미노출).

## [2026-06-16] 메인 히어로 정제 + 카테고리 데코 프리미엄 모노톤 전환
- **배경:** ① 데스크톱 메인 좌우 회색 여백 과다, ② 히어로 헤드라인 "배관공사,"가 좁은 컬럼에서 단어 중간(배관/공사) 분리, ③ 좌상단 로고 그라데이션 애니메이션 변화율이 과해 전문성 저하, ④ 5대 공종 하단 데코가 무지개 5색(주황 2색 중복)으로 맥킨지 톤과 충돌.
- **결정:** ① 히어로 카드 `max-w-[1320px]→1480px`, 하단 통계·푸터 `1240→1400px`로 여백 균형 축소. ② 헤드라인에 `break-keep` + 좌측 카피 컬럼 확대(`1.05fr→1.16fr`, gap 축소)로 "배관공사," 한 줄(40px 유지). ③ 로고 애니메이션 `4s→9s, ease-in-out`(웹/앱/스플래시 3곳). ④ 카테고리 데코를 **프리미엄 모노톤**으로 전환 — 평상시 통일된 네이비 헤어라인(navy/20, 풀 컬럼폭), 호버 시 브랜드 오렌지(`#FF6A00`) 좌→우 채움. 색 구분 역할은 카테고리 텍스트가 담당.
- **기대효과:** 절제된 강조색 사용으로 컨설팅 프리미엄감 강화 및 시선이 헤드라인·CTA로 집중. 헤드라인 가독성/안정성 확보, 로고가 잔잔한 시머로 전문성 제고.
