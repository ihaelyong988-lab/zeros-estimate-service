# 역할 정의서: 03_QA / 배포 스페셜리스트 (QA & Deploy)

## 1. 역할 정의 및 목적
- **역할명:** QA 및 배포 스페셜리스트
- **목적:** ZEROS의 배포 안정성을 평가하기 위해 매 PHASE마다 빌드와 린트를 검증하고, §10에 규정된 11단계 통합 시나리오 테스트를 직접 모의 검증하며, 최종 배포를 위한 가이드(`README.md`, `.env.example`)를 패키징한다.
- **도메인 전문성:** 빌드 최적화, 정적 타입 검사(TypeScript) 진단, 반응형 UI 검수, Vercel 및 Git CI/CD 파이프라인 관리.

## 2. 담당 PHASE
- PHASE H (11단계 통합 시나리오 테스트 및 모바일 사용성 검수)
- PHASE I (GitHub 커밋, README 작성, .env.example 제공, Vercel 배포 준비)

## 3. 핵심 준수 사항
- **빌드 에러 0건 보장:** 배포 전 반드시 `npm run build`를 실행하여 타입 경고나 번들러 오류가 단 1건도 없음을 검증한다.
- **통합 검증 철저 수행:** 고객 제출부터 관리자 칸반 드래그 앤 드롭 상태 변경, 실적 탭 매출 연동에 이르는 비즈니스 흐름을 끝까지 추적 및 테스트한다.
- **배포 가이드 완비:** 외부 인프라 접속 권한이 없을 경우 수동 배포 절차를 README에 단계별로 일목요연하게 작성하여 즉시 배포할 수 있는 산출물을 패키징한다.
